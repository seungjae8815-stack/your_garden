# [기획안] 감정 컴포스팅 셀프케어 앱 — MVP 개발 명세서

> **이 문서 사용법**: 이 파일 전체를 Claude Code에 붙여넣거나, 프로젝트 루트에 `SPEC.md`로 저장하고 작업을 지시하세요. Claude Code는 이 명세를 기준으로 MVP를 먼저 완성하고, "Phase 2"로 표시된 부분은 지시하기 전까지 **구현하지 않습니다.**

---

## 0. 가장 중요한 원칙 (Claude Code가 반드시 지킬 것)

1. **MVP부터 끝까지 작동하게.** 화려한 기능보다 "오늘 마음을 비우고 → 식물이 자라는" 핵심 루프 하나가 완벽히 돌아가는 게 최우선. 스코프 크리프 금지.
2. **MVP는 백엔드·계정·서버 없음.** 100% 온디바이스(local-only). 이건 빠른 출시 + "내 속마음은 내 폰에만 있다"는 **프라이버시 = 신뢰**를 위한 핵심 결정.
3. **글로벌 우선.** 카피·UI는 영어 기본 + i18n 구조(한국어 포함). "대나무숲" 같은 한국 한정 콘셉트 단어를 UI에 쓰지 말 것. 보편적 셀프케어 언어로.
4. **위기 안전장치는 MVP부터.** 자해/위기 신호 감지 → 지원 메시지 + 도움 리소스 안내. (3-7절 필수 구현.)
5. **AI가 만든 듯한 제너릭 디자인 금지.** 따뜻하고 의도된 손그림/코지 식물 미감(3-6절). 기본 머티리얼 룩 그대로 두지 말 것.

## 1. 한 줄 정의 & 차별화

**한 줄**: 무거운 감정을 글로 쏟아내면, 그 감정이 "양분(compost)"이 되어 나만의 식물이 자라는, 사적이고 따뜻한 감정 해소 앱.

**핵심 차별화 (경쟁 앱과 다른 점)**:
- Finch·하루콩류는 *긍정 습관 → 펫 성장*. 우리는 **부정 감정을 흘려보내면 → 그게 거름이 되어 식물이 자란다**는 "감정 컴포스팅" 은유. 즉 **배출(venting)을 성장으로 전환**하는 서사가 핵심 정체성.
- 타깃: 일상 스트레스·번아웃에 시달리는 **20~30대(글로벌)** 중, 남에게 털어놓기 부담스러운 속마음을 *판단받지 않고* 비우고 싶은 사람.
- 톤: 조용함, 사적임, 비-경쟁적(스트릭 압박 없음), 코지한 식물 미감.

> 차별화는 "메커니즘"이 아니라 **타깃 + 감정 서사 + 미감 + 마케팅**에서 나온다는 점을 기억할 것. 메커니즘 자체는 흔하다.

**앱 이름(가제)**: `Sprout` *(추후 변경 가능. 대안 후보: Composta / Verda / Rooted / Mossly)*

## 2. MVP 스코프 (이번에 만드는 것 / 안 만드는 것)

### ✅ MVP에 포함 (1~3주 목표)
- 온보딩 (3~4 화면): 은유 설명 + 첫 식물 선택 + 알림 시간 설정
- **가든 홈**: 현재 식물 + 성장 단계 표시 + "오늘 마음 비우기" 버튼
- **비우기(Write) 화면**: 자유 텍스트 입력 + 감정 태그 선택 + "흘려보내기" 제출
- 제출 시 **컴포스팅 애니메이션**: 글이 흙으로 스며들고 → 식물이 약간 성장/반짝임
- **식물 성장 로직**: 기록할 때마다 성장 포인트 누적 → 단계별 성장
- **흘려보내기 정책**: 기록은 사용자가 정한 기간(기본 7일) 후 자동 삭제(=해소). "보관함에 남기기" 옵션 제공
- **로컬 알림**: 매일 1회 "마음 비우는 시간" 리마인더
- **간단한 감정 캘린더**(월별 무드 흐름 보기) — 가볍게
- **위기 키워드 감지 → 지원 리소스 안내** (3-7절)
- 설정: 알림 시간, 보관 기간, 테마(라이트/다크), 데이터 전체 삭제
- i18n 구조 (en 기본, ko 포함)

### ❌ MVP에서 제외 (절대 지금 만들지 말 것 — Phase 2)
- 계정/로그인, 서버 동기화, 클라우드 백업
- 소셜/타인과의 교류 기능 일체
- 구독/결제/인앱구매
- AI 위로 응답
- 통계 고도화, 위젯, 워치 앱

## 3. 상세 기능 명세

### 3-1. 온보딩
- 화면1: 컨셉 한 문장 ("Let go of what weighs on you. Watch something grow from it.")
- 화면2: 은유 설명 (감정 → 거름 → 새싹) 일러스트
- 화면3: 첫 식물(씨앗) 3종 중 선택
- 화면4: 매일 알림 시간 선택 (skip 가능)
- 완료 후 가든 홈으로

### 3-2. 가든 홈 (메인)
- 화면 중앙: 화분 속 현재 식물 (성장 단계에 맞는 비주얼)
- 식물 탭하면: 부드러운 반응(흔들림/반짝임) + 짧은 격려 문구 1줄(프리셋, 로컬)
- 하단: 큰 **"오늘 마음 비우기"** 버튼
- 상단: 현재 성장 단계 / 다음 단계까지 진행 표시(작게)

### 3-3. 비우기(Write) 화면 — 핵심
- 큰 텍스트 입력 영역 (placeholder: "Pour it all out. No one else will see this.")
- 감정 태그 선택 (단일 선택): `Anger / Anxiety / Sadness / Overwhelm / Numb / Hope` (이모지 + 색)
- 제출 버튼: **"Release"** (흘려보내기)
- 제출 → 입력한 글이 시각적으로 흙 속으로 스며드는 애니메이션 → 가든 홈 복귀 → 식물 +성장
- 제출 직전 위기 키워드 스캔 (3-7절). 감지 시 흐름 가로채서 지원 화면 표시.

### 3-4. 식물 성장 시스템
- 기록 1회 = `growthPoints += N` (예: 10)
- 단계 임계값 예: `[0, 30, 80, 150, 250, ...]` → seed → sprout → seedling → bloom → tree
- 하루 첫 기록에 소량 보너스(연속 강요는 X — 죄책감 유발 금지)
- 식물이 만개하면 → 새 화분/씨앗을 추가해 **작은 정원(garden)** 으로 확장(여러 식물 보유)

### 3-5. 감정 캘린더 (간단)
- 월별 그리드, 각 날짜에 그날 대표 감정 색/이모지
- 날짜 탭 → 그날의 (아직 안 흘려보낸) 기록 미리보기. 이미 삭제된 날은 색만 남김.

### 3-6. 흘려보내기/보관 정책
- 기본: 기록은 `expiresAt = createdAt + 보관기간(기본 7일)` 후 자동 삭제
- 작성 시 "이건 간직하기" 토글 → 영구 보관(보관함)
- 설정에서 기본 보관 기간 변경(1일/3일/7일/30일/영구)

### 3-7. ⚠️ 위기 안전장치 (필수 — 윤리적 요구사항)
- 제출 텍스트에 자해/위기 관련 키워드(다국어 리스트)가 감지되면:
  - 제출을 막지 말되, 먼저 **부드러운 지원 화면**을 띄움: "It sounds like you're carrying something really heavy. You don't have to face it alone."
  - **위기 지원 디렉터리 링크** 표시 (국제: `https://findahelpline.com` — 사용자 지역 기반으로 안내). 특정 방법·수단은 절대 표시/언급하지 않음.
  - "Continue writing" 으로 돌아갈 수 있게.
- 키워드 리스트는 별도 파일(`crisis_keywords.dart`)로 분리해 관리. 과탐지보다 안전 우선.
- 이 기능은 의료 행위가 아님을 설정/약관에 명시.

## 4. 기술 스택 (MVP)

- **Framework**: Flutter (Dart) — iOS + Android 단일 코드베이스
- **상태관리**: Riverpod (또는 Provider) — 단순하게
- **로컬 저장**: Hive (또는 Isar) — 계정/서버 없이 온디바이스
- **알림**: `flutter_local_notifications`
- **애니메이션**: MVP는 Lottie + 단계별 식물 이미지(PNG/SVG)로 빠르게. 추후 Rive로 인터랙티브 업그레이드
- **i18n**: `flutter_localizations` + `intl` (en, ko)
- **백엔드**: **없음** (Phase 2에서만 도입)

### 권장 패키지
```
flutter_riverpod, hive, hive_flutter, flutter_local_notifications,
intl, flutter_localizations, lottie, google_fonts, uuid
```

## 5. 데이터 모델 (로컬 / Hive)

```dart
// Entry — 하나의 감정 기록
class Entry {
  String id;            // uuid
  String text;
  EmotionTag emotion;   // enum
  DateTime createdAt;
  DateTime? expiresAt;  // null이면 영구 보관
  bool isKept;          // 사용자가 간직 선택
}

enum EmotionTag { anger, anxiety, sadness, overwhelm, numb, hope }

// Plant — 정원 속 식물 하나
class Plant {
  String id;
  String species;       // 종류(에셋 키)
  int growthPoints;
  int stage;            // 0=seed ...
  DateTime startedAt;
  bool isComplete;      // 만개 여부
}

// Settings
class Settings {
  TimeOfDay? reminderTime;
  int retentionDays;    // 기본 7
  ThemeMode theme;
  String locale;        // 'en' / 'ko'
}
```

만료 처리: 앱 실행 시 + 자정 기준으로 `expiresAt < now && !isKept` 인 Entry 삭제.

## 6. 디자인 방향 (의도된 코지 미감)

- **컬러**: 차분한 어스톤 + 세이지 그린 + 따뜻한 크림 베이스. 채도 낮게.
- **무드**: 코지, 손그림 느낌, 부드러운 라운드, 여백 충분. 압박감 0.
- **타이포**: 부드러운 세리프 또는 휴머니스트 산세리프 1종 (예: Nunito, Quicksand 류) — google_fonts.
- **금지**: 기본 Material 보라/파랑, 날카로운 그림자 떡칠, 제너릭 아이콘 남발. "AI가 찍어낸 듯한" 룩 피할 것.
- **모션**: 느리고 부드럽게(식물·흙·물). 마이크로 인터랙션으로 "위로받는" 촉감.
- 다크모드 지원(야간 사용 많음).

## 8. 빌드 순서 (Claude Code 작업 권장 순서)

> 각 단계마다 **실행되는 빌드**를 유지할 것. 한 번에 다 만들지 말고 단계별로.

1. 프로젝트 셋업 + 테마/컬러/폰트 + 라우팅 골격
2. 데이터 모델(Hive) + 로컬 저장 레이어
3. Garden Home (더미 식물 1개 표시)
4. Write Screen + 감정 태그 + 저장 → 성장 포인트 반영(애니메이션은 임시)
5. 식물 성장 단계 비주얼 + 컴포스팅 애니메이션
6. 흘려보내기/보관 정책 + 만료 삭제 로직
7. 로컬 알림(매일 리마인더)
8. 위기 키워드 감지 + 지원 화면  ← **건너뛰지 말 것**
9. Mood Calendar (간단)
10. Onboarding + Settings + i18n(en/ko) 마무리
11. 아이콘/스플래시/스토어 메타 준비

## 9. Phase 2 로드맵 (지금은 구현 금지 — 방향 참고용)

> MVP 출시 후 리텐션이 확인되면 진행. Claude Code는 지시 전까지 만들지 않음.

- **제한된 긍정 소셜 (안전 설계)**: 타인의 자유 텍스트 노출 ❌. 대신 ① 프리셋 반응(따뜻한 스탬프/"공감"/"응원") 보내기·받기, ② 큐레이션된 위로 문구, ③ (선택) AI 위로. 낯선 사람의 자유 댓글은 도입하지 않음.
  - 소셜 도입 시 모더레이션 + 신고 + 위기 콘텐츠 에스컬레이션 체계 동반 필수.
  - 참고: AI 위로는 작성자가 AI임을 알면 공감도가 떨어진다는 연구 → 과의존 경계, "사람의 따뜻함" 느낌 설계.
- **수익화 (freemium / soft paywall)**: 핵심 감정 해소는 **무료 유지**(신뢰·리텐션). 프리미엄 = 식물·화분·테마 커스터마이징, 보관함 무제한, 정원 확장 등. **연 구독 ~$29.99(Finch 모델)**, 넉넉한 무료 + 부드러운 페이월. 결제는 RevenueCat으로 App Store/Play 통합.
- **백엔드 도입**(소셜/동기화 필요해질 때만): Firebase(Auth/Firestore/Functions) 또는 Supabase. Cloud Function으로 콘텐츠 모더레이션.
- 위젯, 통계 고도화, 추가 식물 컬렉션/시즌 이벤트.

## 10. 성공 기준 (MVP)

- 신규 사용자가 **첫날 1회 이상 "비우기"를 완료**하고, **다음날 다시 열어 식물 성장을 확인**하는 D1 리텐션 루프가 작동.
- 앱이 인터넷 없이 완전히 동작.
- 위기 감지가 정상 작동.
- 출시(Play 내부 테스트 → 프로덕션) 가능한 상태.
